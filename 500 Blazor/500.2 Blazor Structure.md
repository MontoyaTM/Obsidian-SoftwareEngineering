# Program.cs

```run-csharp
public class Program
{
    public static void Main(string[] args)
    {
        var builder = WebApplication.CreateBuilder(args);

        // Add services to the container.
        builder.Services.AddRazorComponents();

        var app = builder.Build();

        // Configure the HTTP request pipeline.
        if (!app.Environment.IsDevelopment())
        {
            app.UseExceptionHandler("/Error");
            // The default HSTS value is 30 days. You may want to change this for production scenarios, see https://aka.ms/aspnetcore-hsts.
            app.UseHsts();
        }

        app.UseStatusCodePagesWithReExecute("/not-found", createScopeForStatusCodePages: true);
        app.UseHttpsRedirection();

        app.UseAntiforgery();

        app.MapStaticAssets();
        app.MapRazorComponents<App>();

        app.Run();
    }
}
```

### app.Run( )

Runs an application  and blocks the calling thread until host shutdown

### app.MapRazorComonents«App»( )

Maps the page components defined in the specified App to the given assembly and renders the component specified by App when the route matches.

- Returns: <span style="color:rgb(132, 255, 0)">RazorComponentsEndpointConventionBuilder</span> that can be used to further configure the API.

``` run-csharp
<!DOCTYPE html>
```

In a **Blazor Web App**, `App.razor` is essentially the **entry point of your UI pipeline**—it controls how <span style="color:rgb(132, 255, 0)">routing</span>, <span style="color:rgb(132, 255, 0)">layouts</span>, and <span style="color:rgb(132, 255, 0)">authorization</span> are handled at runtime.

At runtime `App.razor` uses the <span style="color:rgb(132, 255, 0)">«Router»</span> component to:

- Scan assemblies for <span style="color:rgb(181, 98, 249)">@page</span> directives
- Match the current URL to a component
- Renders the correct page

> App.razor
    → Router
       → RouteView
          → MainLayout
             → Your Page


# Routes.razor

`Routes.razor` is responsible for **defining how pages are routed and rendered** in your app. 

- Controls **SSR vs Interactive behavior** . 
- Acts as the **bridge between navigation and UI rendering**

### Key responsibilities

- Maps URLs → Razor components (`@page`)
- Applies **render mode** (SSR, Interactive, etc.)
- Handles:
    - Found routes
    - Not found routes
    - Authorization (if used)


``` run-csharp
<Router AppAssembly="typeof(Program).Assembly" NotFoundPage="typeof(Pages.NotFound)">
    <Found Context="routeData">
        <RouteView RouteData="routeData" DefaultLayout="typeof(Layout.MainLayout)" />
        <FocusOnNavigate RouteData="routeData" Selector="h1" />
    </Found>
</Router>
```


# MainLayout.razor

`MainLayout.razor` defines the **shared UI structure** that wraps your pages.

### Key responsibilities

- Provides consistent layout:
    - Sidebar
    - Topbar
    - Footer
- Renders page content using `@Body`


``` run-csharp
@inherits LayoutComponentBase

<div class="page">
    <div class="sidebar">
        <NavMenu />
    </div>

    <main>
        <div class="top-row px-4">
            <a href="https://learn.microsoft.com/aspnet/core/" target="_blank">About</a>
        </div>

        <article class="content px-4">
            @Body
        </article>
    </main>
</div>
```
