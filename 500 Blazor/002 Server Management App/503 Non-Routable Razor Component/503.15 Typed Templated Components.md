# RepeaterComponent.razor

A **generic, virtualized repeater** that uses templates (`RenderFragment`) to render items, with optional header and footer support.

``` run-csharp
@typeparam TItem

@Header
@if(Items is not null && Items.Count > 0 &&  Row is not null)
{
    <Virtualize Items="Items" Context="item">
        @Row(item)
    </Virtualize>
}
@Footer

@code {
    [Parameter]
    public List<TItem>? Items { get; set; }

    [Parameter]
    public RenderFragment<TItem>? Row { get; set; }
    
    [Parameter]
    public RenderFragment? Header { get; set; }
    
    [Parameter]
    public RenderFragment? Footer { get; set; }
}
```

- `@typeparam TItem`
	- Makes the component **generic** 
	- `TItem` represents the type of data in the list

- `@if(Items is not null && Items.Count > 0 && Row is not null)`
	- Main Rendering Logic
	- `Items` exists
	- List is not empty
	- A row template (`Row`) is provided
	- Prevents runtime errors and empty rendering.

- `<Virtualize Items="Items" Context="item">`
	- Only renders **visible items on screen**
	- Improves performance for large lists
	- Instead of rendering 10,000 items, it might render only ~20 at a time.

- `@Row(item)`
	- Calls a **template passed from the parent**
	- Defines how each item should look

# ServerListComponent.razor

##### // Generic template component
``` run-csharp
<ul class="list-unstyled">
    <RepeaterComponent Items="servers">
        <Row Context="server">
            <ServerComponent Server="server"></ServerComponent>
        </Row>
    </RepeaterComponent>
</ul>
```

``` run-csharp
@code {
    [Parameter]
    public string? CityName { get; set; }

    [Parameter]
    public string SearchFilter { get; set; } = "";

    private List<Server>? servers;

    override protected void OnParametersSet()
    {
        if(string.IsNullOrWhiteSpace(SearchFilter))
        {
            servers = ServersRepository.GetServersByCity(CityName ?? "Toronto");
        }
        else
        {
            servers = ServersRepository.SearchServer(SearchFilter);
        }
    }
}
```

### What’s happening here:

- `<ul class="list-unstyled">`
    - A standard HTML unordered list with no default styling.

- `<RepeaterComponent Items="servers">`
    - A **custom reusable component** that loops over a collection (`servers`).
    - Think of it like a `foreach` in UI form.

- `<Row Context="server">`
    - Defines how each item should be rendered.
    - `Context="server"` means each item in `servers` is available as `server`.

- `<ServerComponent Server="server" />`
    - For each server, it renders another component (`ServerComponent`).
    - Passes the current `server` object as a parameter.
