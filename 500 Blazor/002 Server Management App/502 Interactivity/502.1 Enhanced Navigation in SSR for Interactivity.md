#### "Request and response model"

The blazor.web.js script is **one of the most critical pieces of a modern Blazor Web App (especially .NET 8/9/10)**. It acts as the **runtime bridge between your C# components and the browser**. Without it, your app is just **static HTML**.. 

- Boots the Blazor runtime in the browser
- Initializes the Blazor environment
- Connects your rendered HTML to interactive components
- Starts the app lifecycle

This script is responsible for making these work:

- `@onclick`, `@bind`, `EventCallback`
- Component re-rendering
- State updates (like your `LayoutState`, sidebar toggle, theme switching)
- Forms and validation

In .NET 8+ Blazor Web Apps, this script enables:

- **Interactive Server**
- **Interactive WebAssembly**
- **Auto mode**

Manages SignalR (for Interactive Server):

- Opens a **SignalR connection**
- Syncs UI changes between browser ↔ server
- Sends events (clicks, inputs) to your server-side components

Enables JS interop:

- C# → JavaScript calls
- JavaScript → C# callbacks

##### // Retrieve data from form
``` run-csharp
@code {
    [SupplyParameterFromForm]
    private Server? server { get; set; } //new Server { Name = "Server1", City = "Toronto" };
    override protected void OnParametersSet()
    {
        server ??= new Server { Name = "Server1", City = "Toronto" };
    }

    private void ChangerServerStatus()
    {
        if (server is not null)
        {
            server.IsOnline = !server.IsOnline;
        }
    }
}
```
