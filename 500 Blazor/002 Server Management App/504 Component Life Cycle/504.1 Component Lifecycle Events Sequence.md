# When does a component render/re-render?

In Blazor, a component renders (or re-renders) whenever its **state changes in a way that affects the UI**. The framework automatically decides _when_ to update the DOM based on certain triggers.

``` run-csharp
<ChildComponent Value="@someValue" />
```

When `someValue` changes:

- Child’s `OnParametersSet()` runs
- **Child re-renders**

Any UI event handled by the component triggers a re-render automatically.

``` run-csharp
<button @onclick="Increment">Click</button>

@code {
    int count = 0;

    void Increment()
    {
        count++;
    }
}
```

After `Increment()` runs:

- Blazor **automatically re-renders**

You can force a re-render manually:

``` run-csharp
StateHasChanged();
```

Use this when:

- State changes **outside normal UI events**
- Example: timers, background tasks, or external callbacks


If a **cascading value** changes:

- All consuming components re-render

When a child invokes an `EventCallback`:

- Parent updates state
- Parent re-renders
- Updated parameters flow back down → child re-renders


A Blazor component re-renders when:

- ✅ It first loads
- ✅ Its **parameters change**
- ✅ A **UI event** occurs (`@onclick`, etc.)
- ✅ `StateHasChanged()` is called
- ✅ **Async lifecycle work completes**
- ✅ **Cascading values change**
